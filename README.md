### 🔐 IAM Dashboard — Sistema de Gestão de Identidades

[![Repo](https://img.shields.io/badge/GitHub-IAM_Dashboard-00ffc8?style=for-the-badge&logo=github&logoColor=black)](https://github.com/carlos-silva-ads/iam-dashboard)
![Python](https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white)
![Flask](https://img.shields.io/badge/Flask-000000?style=flat-square&logo=flask&logoColor=white)
![SQLite](https://img.shields.io/badge/SQLite-003B57?style=flat-square&logo=sqlite&logoColor=white)
![REST API](https://img.shields.io/badge/REST_API-FF6C37?style=flat-square&logo=postman&logoColor=white)

</div>

> Sistema web que simula o **ciclo de vida completo de identidades corporativas**, o mesmo conceito aplicado em plataformas enterprise como **SailPoint IdentityNow**.

**O que implementa:**

- ✅ **Provisionamento** de identidades com cargo, departamento e email
- ✅ **Lifecycle Management** — ativação e desativação de acessos em tempo real
- ✅ **Deprovisioning** com rastreamento completo
- ✅ **Audit Trail imutável** — todo evento registrado (CREATE · ACTIVATE · DEACTIVATE · DEPROVISION)
- ✅ **REST API** com GET · POST · PATCH · DELETE e status codes corretos
- ✅ **SQL puro** sem ORM com proteção contra SQL Injection

```
📁 Conceitos aplicados: IGA · Identity Lifecycle · Zero Trust · Audit Log · REST API
```

### 🛡️ Auth Security Lab — Autenticação com MFA & LDAP -🔄 Em andamento

[![Repo](https://img.shields.io/badge/GitHub-Auth_Security_Lab-00b4ff?style=for-the-badge&logo=github&logoColor=black)](https://github.com/carlos-silva-ads/auth-security-lab)
![Java](https://img.shields.io/badge/Java-ED8B00?style=flat-square&logo=openjdk&logoColor=white)
![MFA](https://img.shields.io/badge/MFA-FF4444?style=flat-square&logo=authy&logoColor=white)
![Security](https://img.shields.io/badge/Security-00C853?style=flat-square&logo=shield&logoColor=white)

</div>

> Sistema de autenticação corporativa em **Java** com múltiplas camadas de segurança, simulando o que **CyberArk** e **Ping Identity** fazem em produção.

**O que implementa:**

- ✅ **MFA com TOTP** — compatível com Google Authenticator
- ✅ **Rate limiting** — bloqueio após 3 tentativas falhas (Least Privilege)
- ✅ **Simulação de LDAP** — estrutura de diretório DN · CN · OU
- ✅ **Audit Log** em arquivo físico no formato padrão Unix
- ✅ **Princípio Zero Trust** — nunca confie, sempre verifique

```
📁 Conceitos aplicados: PAM · MFA · LDAP · Zero Trust · Rate Limiting · Audit Trail
```
